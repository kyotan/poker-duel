# POKER DUEL

5枚の手札を高速で更新し、成立したポーカー役を直接発動して相手のHPを削る、リアルタイムカードバトルです。

現在はCPU対戦に加え、同じWi-Fi内の2台で遊べるLANオンラインPvPを実装済みです。PC↔PC、PC↔スマートフォン、スマートフォン↔スマートフォンで、交換・役攻撃・スキル・再戦まで対戦できます。

## ゲームをプレイ

[POKER DUELをブラウザで起動](https://kyotan.github.io/poker-duel/)

## 現在遊べる内容

- START後5秒で双方のカードを一斉公開
- カード公開から最大3分。タイムアップ時はHPが多い側の勝利、同HPならDRAW
- 残り10秒の数字カウントダウンと、画面全体の赤いパルス演出
- プレイヤーごとの独立52枚と、返却・全体シャッフル・即時補充
- 1〜5枚の自由交換
- 全9種の役と、同じ手札にある複数の発動候補
- 役ボタンによる即時攻撃と2〜5段のカード演出、強い着弾・被弾フィードバック
- 相手を待たずに行動するCPUアクティブバトル
- 15秒ごとに出現し、未取得なら10秒で消えるスキル争奪
- HEAL +20／STOP／SHUFFLE／STEAL／BLOCK
- 日本語／英語切替（右上、端末ごとに保存）
- 同じWi-Fi内での招待制オンラインPvP

## 同じWi-Fiでオンライン対戦する

このLAN検証版はCloudflareの契約やアカウントなしで試せます。ゲームを起動するPCを1台用意し、対戦に使う端末をすべて同じWi-Fiへ接続してください。

### 1. PCでゲームを起動する

初回だけ次を実行します。

```sh
npm install
```

続いてLAN対戦版を起動します。

```sh
npm run dev:lan
```

画面に表示される`Network`のURLを確認します。たとえば次のようなアドレスです。

```text
http://192.168.11.3:5173/
```

`http://localhost:5173/`や`http://127.0.0.1:5173/`では、ほかのPCやスマートフォンから参加できません。ホスト側も含め、対戦する両方の端末で同じ`Network` URLを開いてください。

### 2. 部屋を作って招待する

1. 最初の画面で`ONLINE PvP`を選ぶ
2. 表示名を入力して`部屋を作る`を押す
3. 相手へQRコード、招待リンク、または6文字コードを渡す
4. 相手も同じ`Network` URLを開き、表示名を入力してリンクまたはコードから参加する
5. 2人とも`START`を押す
6. 5秒のカウントダウン後にカードが公開され、対戦が始まる

PC↔PC、PC↔スマートフォンのほか、PCをゲームのホストとして起動したままにすればスマートフォン↔スマートフォンでも遊べます。ホストPCは対戦中にスリープさせたり、`npm run dev:lan`を停止したりしないでください。

### 接続できない場合

- OSのファイアウォールでNode.jsからの接続を許可する
- すべての端末が同じ通常のWi-Fiへ接続されているか確認する
- ホテルや店舗のゲストWi-Fiは、端末同士の通信を禁止している場合があるため避ける
- ルーターの`AP isolation`、`プライバシーセパレーター`、`端末間通信の禁止`が有効なら解除する
- URLが`localhost`や`127.0.0.1`ではなく、`192.168...`などから始まっているか確認する

LAN版の部屋と試合状態はホストPCのメモリだけに保存されます。ゲームを停止すると部屋は消え、対戦中の試合も復元できません。

> このLAN検証版はGitHub Pages公開版とは別の起動方法です。GitHub PagesのURLだけで離れた場所の相手とオンライン対戦することはできません。
